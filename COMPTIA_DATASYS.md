# CompTIA DataSys+ DS0-001 Certification Study Guide

> Vendor-neutral database administration certification - Systems design & development focus

## Exam Overview

| Field | Details |
|-------|---------|
| **Exam Code** | DS0-001 |
| **Version** | 4.0 |
| **Questions** | 90 maximum |
| **Length** | 90 minutes |
| **Passing Score** | 750 (on a scale of 100-900) |
| **Languages** | English |
| **Prerequisites** | 2-3 years database administration experience |

---

## Exam Objectives (Domains)

### Domain 1.0: Database Fundamentals (24%)

#### 1.1 - Compare and contrast database structure types
- Relational databases
- NoSQL databases
- Document databases
- Key-value stores
- Graph databases
- Time-series databases

#### 1.2 - Given a scenario, apply database design principles
- Normalization (1NF, 2NF, 3NF)
- Entity-Relationship (ER) modeling
- Primary keys and foreign keys
- Index design
- Data types selection

#### 1.3 - Explain database objects and their uses
- Tables
- Views
- Stored procedures
- Functions
- Triggers
- Indexes
- Constraints

---

### Domain 2.0: Database Deployment (18%)

#### 2.1 - Given a scenario, plan database deployment
- Capacity planning
- Storage requirements
- High availability considerations
- Backup strategy
- Disaster recovery plan

#### 2.2 - Compare and contrast database deployment methods
- On-premises deployment
- Cloud deployment (IaaS, PaaS, SaaS)
- Hybrid deployment
- Containerized deployment

#### 2.3 - Given a scenario, configure database settings
- Memory configuration
- CPU configuration
- Storage configuration
- Network settings
- Security settings

#### 2.4 - Explain database connectivity options
- ODBC
- JDBC
- Native drivers
- REST APIs
- GraphQL

---

### Domain 3.0: Database Management and Maintenance (22%)

#### 3.1 - Given a scenario, perform database backup and restore operations
- Full backups
- Incremental backups
- Differential backups
- Point-in-time recovery
- Backup verification

#### 3.2 - Perform database maintenance tasks
- Index maintenance
- Statistics updates
- Database tuning
- Query optimization
- Performance monitoring

#### 3.3 - Explain data movement operations
- ETL (Extract, Transform, Load)
- Data import/export
- Data migration
- Replication
- Data warehousing concepts

#### 3.4 - Manage database users and permissions
- User creation
- Role management
- Permission assignment
- Authentication methods
- Password policies

---

### Domain 4.0: Data and Database Security (18%)

#### 4.1 - Compare and contrast data encryption methods
- Transparent Data Encryption (TDE)
- Column-level encryption
- Application-level encryption
- Encryption at rest
- Encryption in transit
- Key management

#### 4.2 - Implement database security measures
- Access control lists
- Row-level security
- Column-level security
- Audit logging
- Vulnerability assessment

#### 4.3 - Explain compliance and regulatory requirements
- GDPR compliance
- HIPAA compliance
- PCI DSS compliance
- Data retention policies
- Data masking/anonymization

#### 4.4 - Secure database communications
- SSL/TLS configuration
- Certificate management
- Firewall rules
- Network segmentation

---

### Domain 5.0: Business Continuity (18%)

#### 5.1 - Compare and contrast high availability options
- Active-passive clustering
- Active-active clustering
- Database mirroring
- Always On availability groups
- Replication (synchronous vs asynchronous)

#### 5.2 - Explain disaster recovery concepts
- Recovery Point Objective (RPO)
- Recovery Time Objective (RTO)
- Failover strategies
- Site redundancy
- Cloud disaster recovery

#### 5.3 - Implement database backup and restore for recovery
- Backup strategies
- Restore procedures
- Testing backups
- Disaster recovery testing

#### 5.4 - Monitor database health and performance
- Performance metrics
- Alerting
- Capacity planning
- Trend analysis

## Database Systems Covered

This certification is vendor-neutral but covers concepts applicable to:
- **PostgreSQL** (Open source - used in Supabase)
- **MySQL/MariaDB**
- **Microsoft SQL Server**
- **Oracle Database**
- **MongoDB** (NoSQL)
- **Amazon RDS**
- **Azure SQL**
- **Google Cloud SQL**
