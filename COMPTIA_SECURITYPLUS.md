# CompTIA Security+ SY0-701 Certification Study Guide

> Vendor-neutral cybersecurity certification - Entry-level security certification

## Exam Overview

| Field | Details |
|-------|---------|
| **Exam Code** | SY0-701 |
| **Version** | 5.0 |
| **Questions** | 90 maximum |
| **Length** | 90 minutes |
| **Passing Score** | 750 (on a scale of 100-900) |
| **Languages** | English, Japanese, Portuguese |
| **Certification Renewal** | 3 years |

---

## Exam Objectives (Domains)

### Domain 1.0: General Security Concepts (12%)

#### 1.1 - Compare and contrast various types of security controls
- Technical controls
- Physical controls
- Administrative controls
- Preventive controls
- Detective controls
- Corrective controls
- Deterrent controls
- Compensating controls
- Directive controls

#### 1.2 - Summarize fundamental security concepts
- Confidentiality
- Integrity
- Availability (CIA Triad)
- Non-repudiation
- Authentication, Authorization, Accounting (AAA)
- Zero Trust
- Deception/disruption technology

#### 1.3 - Explain the importance of change management processes
- Impact of unmanaged changes on security
- Change management workflows
- Documentation requirements

#### 1.4 - Explain importance of appropriate cryptographic solutions
- Cryptography for confidentiality
- Hashing for integrity
- Digital signatures for non-repudiation
- Encryption types (symmetric, asymmetric)

---

### Domain 2.0: Threats, Vulnerabilities, and Mitigations (22%)

#### 2.1 - Compare and contrast common threat actors and motivations
- Nation-state actors
- Hacktivists
- Insider threats
- Organized crime
- Script kiddies
- Advanced Persistent Threats (APT)

#### 2.2 - Explain common threat vectors and attack surfaces
- Phishing
- Malware (ransomware, trojans, worms, viruses)
- Social engineering
- Unpatched vulnerabilities
- Misconfigurations
- Zero-day exploits

#### 2.3 - Compare and contrast different types of vulnerabilities
- Software vulnerabilities
- Hardware vulnerabilities
- Operating system vulnerabilities
- Network vulnerabilities
- Physical vulnerabilities

#### 2.4 - Summarize techniques used in vulnerability assessments
- Passive scanning
- Active scanning
- Vulnerability scoring (CVSS)
- False positive/negative analysis

---

### Domain 3.0: Security Architecture (26%)

#### 3.1 - Explain the concepts of security architecture
- Defense in depth
- Network segmentation
- Air gapping
- Physical security integration

#### 3.2 - Compare and contrast security implications of different architecture models
- Cloud models (IaaS, PaaS, SaaS)
- On-premises
- Hybrid environments
- Zero Trust architecture

#### 3.3 - Summarize secure infrastructure design
- Network appliances (firewalls, IDS/IPS, proxies)
- Virtual private networks (VPNs)
- Network access control (NAC)
- DNS security

#### 3.4 - Implement secure networking protocols
- TLS/SSL
- IPsec
- SSH
- SNMPv3

---

### Domain 4.0: Operations and Incident Response (22%)

#### 4.1 - Explain the purpose and use of playbooks and playbooks
- Incident response procedures
- Incident categories
- Escalation paths

#### 4.2 - Given a scenario, use appropriate data sources to support an investigation
- Log sources (network, system, application)
- SIEM systems
- Packet captures
- Memory dumps

#### 4.3 - Given a scenario, implement an incident response plan
- Containment strategies
- Eradication
- Recovery
- Post-incident activities

#### 4.4 - Explain key concepts of forensics
- Chain of custody
- Evidence preservation
- Legal hold
- Forensic tools

---

### Domain 5.0: Governance, Risk, and Compliance (18%)

#### 5.1 - Compare and contrast various types of controls
- Preventive vs detective vs corrective
- Technical vs administrative vs physical

#### 5.2 - Summarize elements of effective security governance
- Security policies
- Standards
- Procedures
- Guidelines
- Roles and responsibilities

#### 5.3 - Explain the elements of the risk management process
- Risk identification
- Risk assessment (qualitative, quantitative)
- Risk response
- Risk monitoring

#### 5.4 - Explain the importance of applicable regulations and compliance
- GDPR
- HIPAA
- PCI DSS
- SOX
- Industry-specific regulations
